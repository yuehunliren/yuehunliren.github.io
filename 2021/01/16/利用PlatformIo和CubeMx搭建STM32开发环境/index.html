利用PlatformIo和CubeMx搭建STM32开发环境

2021-01-16- 2021-01-17
-
-STM32 -PlatformIo -CubeMX
yuehunliren

前言介绍

由于不太喜欢MDK的**==自动补全==**和它丑陋的颜值，在发现Vscode上的PlatformIo插件时，我立马就爱上了它，先不说它支持非常多的开发平台，比如说Arduino，STM32,ESP32,ESP8266等，它支持的开发框架也是多种多样。简直就是一个全方位，多平台，多框架式的嵌入式IDE,并且还支持编译下载，调试功能，自带的串口助手也是简直不要太爽。再配合图形化配置软件STM32CubeMx，开发STM32的舒适度将再上一个档次！下面将介绍一下怎么用二者进行配合使用

PlatformIo

  1. 在Vscode拓展中搜索PlatformIo，下载后，Vscode左边侧边栏会出现下图的蚂蚁的标志
  1. 点开后在上面新建工程

  1. 创建工程的名称，此时要注意，工程的名称要记住，等会我们还会使用到，Borad我选择STM32F103RC,读者根据自己手上的板子来选择就行了，Framework我们选择SMT32Cube.

，把Location的默认关掉，使用自己定义的目录地址，我是将这个工程保存在我的D:/STM32STC51/PlatformIo目录下面，等会这个目录会在使用CubeMX时用到。完成这一后点击Finish,等待一段时间后，工程也就创建完毕了

CubeMx配置阶段

  1. 新建项目

  2. 选择 MCU 型号，我的是STM32F103RCT

  3. 配置 Pinout & Configuration

    • 配置 RCC（选外部 / 内部时钟，视情况可略）

    • 配置 SYS（将 DEBUG 选项由 No Debug 修改为 Serial Wire）

  4. 配置 Clock Configuration

  5. 配置 Project Manager

    1. Project 页面
      填写项目名称 （Project Name） 修改为mypro(必须和之前的 PlatformIo里的工程名一样)
      修改项目路径 （Project Location） 修改为D:/STM32STC51/PlatformIo（也和之前的一样)
      将工具链（Toolchain / IDE）修改为 Other Toolchains
      ==将Application Structure改成Basic==(这一步是我之前一直没配置成功的关键！)
  1. Code Generator 页面
    将软件包选项（STM32Cube Firmware Library Package）选择为 Copy only the necessary library files
    在文件生成选项（Generated files）勾选 Generate peripheral initialization as a pair of ‘.c/.h’ files per peripheral
    终于配置完成了，我们点击右上角 Generate Code 生成代码吧。

Vscode中的操作

  1. 打开项目中 platformio.ini 文件，添加如下几行

    [platformio]
    include_dir=Inc
    src_dir=Src

    在framework = stm32cube后面添加下面两行，使用jlnk对项目进行下载和调试（如果你是stlink，换成stlink就行）

    debug_tool = jlink

    upload_protocol =jlink

  2. 这里是因为 PlatformIO 与 CubeMX 默认生成的框架文件夹不一样，为了兼容性，我们顺从 CubeMX.

  3. 可以将项目中的 include 文件夹删了。而因为 Windows 文件命名不区分大小写，所以 src 文件夹顺理成章变为 Src.

最后一步

项目中， .c 存放于 Src 文件夹中，.h 在 Inc 中。
只要在 /* USER CODE BEGIN */ 与 /* USER CODE END */ 之间的代码，后续从 CubeMX 生成的过程中，都将得以保留，不会被覆盖掉。

PlatformIO 可以用快捷键 Ctrl + Alt + B 编译，用 Ctrl + Alt + U 编译并上传，按 F5 开启调试，接下来就是开始享受的时刻！
